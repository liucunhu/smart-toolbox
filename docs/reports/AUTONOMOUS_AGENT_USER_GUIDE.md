## 系统概述

自主智能体系统是 Smart-Toolbox 的核心 AI 驱动自动化平台，能够：
- 🎯 **自动任务规划** - 将模糊目标分解为具体可执行的任务
- 🧠 **强化学习优化** - 通过 Q-Learning 不断优化决策策略
- 🌐 **跨平台知识迁移** - 将在一个平台学到的经验应用到其他平台
- 🤖 **多智能体协作** - 不同类型的智能体协同完成复杂任务

## 快速开始

### 1. 访问监控系统

打开浏览器访问：`http://localhost:3002/autonomous-agent-monitor`

### 2. 注册智能体

点击右上角的 **"注册智能体"** 按钮，选择需要的智能体类型：

#### 推荐的最小配置
```
✅ research (研究智能体) - 用于热点挖掘和选题研究
✅ content_generation (内容生成智能体) - 用于文章创作
✅ distribution (分发智能体) - 用于多平台发布
```

#### 完整配置（可选）
```
✅ planning (规划智能体) - 用于内容大纲规划
✅ compliance_check (合规检查智能体) - 用于违禁词检测
✅ image_generation (图片生成智能体) - 用于封面图生成
✅ nurturing (养号智能体) - 用于账号养护
```

### 3. 查看智能体列表

注册后，智能体会显示在列表中，包括：
- 智能体类型
- 当前状态（空闲/忙碌）
- 能力标签
- 完成任务数
- 负载情况
- 最后心跳时间

## 智能体类型说明

| 智能体类型 | 英文标识 | 主要职责 | 典型应用场景 |
|-----------|---------|---------|------------|
| 研究智能体 | `research` | 热点挖掘、趋势分析、选题研究 | 发现热门话题、分析竞品内容 |
| 规划智能体 | `planning` | 内容大纲、结构规划 | 制定文章框架、设计内容流程 |
| 内容生成智能体 | `content_generation` | 文章撰写、文案创作 | 自动生成文章内容、营销文案 |
| 合规检查智能体 | `compliance_check` | 违禁词检测、内容审核 | 确保内容符合平台规范 |
| 图片生成智能体 | `image_generation` | 封面图生成、配图制作 | 自动生成文章配图和封面 |
| 分发智能体 | `distribution` | 多平台发布、定时推送 | 一键发布到头条、抖音等平台 |
| 养号智能体 | `nurturing` | 账号活跃、互动操作 | 浏览视频、点赞评论、关注账号 |
| 通用智能体 | `general` | 处理未分类的通用任务 | 临时性、非特定类型的任务 |

## 使用流程

### 场景 1：内容创作工作流

#### 步骤 1：输入目标
在"任务规划"面板中输入：
```
创作一篇关于AI的文章并发布到头条
```

#### 步骤 2：设置优先级
拖动滑块设置优先级（1-10），建议设置为 7-8

#### 步骤 3：分解目标
点击 **"分解目标"** 按钮

系统会自动分解为以下任务：
```
步骤 1: 创作一篇关于AI的文章并发布到头条 - topic_research
        智能体类型: research
        
步骤 2: 创作一篇关于AI的文章并发布到头条 - outline_generation
        智能体类型: planning
        
步骤 3: 创作一篇关于AI的文章并发布到头条 - content_writing
        智能体类型: content_generation
        
步骤 4: 创作一篇关于AI的文章并发布到头条 - compliance_check
        智能体类型: compliance_check
        
步骤 5: 创作一篇关于AI的文章并发布到头条 - image_generation
        智能体类型: image_generation
        
步骤 6: 创作一篇关于AI的文章并发布到头条 - publish
        智能体类型: distribution
```

#### 步骤 4：执行任务（待实现）
⚠️ **当前状态**：任务分解已完成，但自动执行功能正在开发中

---

### 场景 2：强化学习调优

#### 使用 Q-Learning 优化决策

1. **输入状态**：例如 `"high_traffic_time"`
2. **输入可用动作**：例如 `"publish_now,schedule_later,draft_only"`
3. **点击"选择动作"**：系统会根据历史经验选择最优动作
4. **查看 Q 值分布**：了解每个动作的评估价值
5. **衰减探索率**：随着经验积累，逐渐减少随机探索

---

### 场景 3：跨平台知识迁移

#### 将今日头条的经验迁移到抖音

1. **选择源平台**：今日头条
2. **选择目标平台**：抖音
3. **选择知识类型**：最佳实践 / 模式规则
4. **点击"迁移知识"**：系统会自动适配并迁移经验

迁移内容包括：
- 发布时间规律
- 内容风格偏好
- 互动技巧
- 热门标签使用

## API 端点参考

### 智能体管理

```bash
# 注册智能体
POST /api/v1/agents/register
{
  "agent_type": "content_generation",
  "capabilities": ["article_writing", "copywriting"]
}

# 获取智能体列表
GET /api/v1/agents/list?agent_type=content_generation&status=idle

# 获取系统状态
GET /api/v1/agents/system/status
```

### 任务规划

```bash
# 分解目标
POST /api/v1/agents/autonomous/planning/decompose
{
  "description": "创作一篇关于AI的文章",
  "priority": 8
}

# 评估不确定性
POST /api/v1/agents/autonomous/planning/assess-uncertainty
{
  "context": {"platform": "toutiao", "account_id": 123}
}
```

### 强化学习

```bash
# 选择动作
POST /api/v1/agents/autonomous/learning/choose-action
{
  "state": "high_traffic_time",
  "available_actions": ["publish_now", "schedule_later"]
}

# 更新 Q 值
POST /api/v1/agents/autonomous/learning/update-q-value
{
  "state": "s1",
  "action": "a1",
  "reward": 0.8,
  "next_state": "s2"
}

# 获取 Q 表统计
GET /api/v1/agents/autonomous/learning/q-table
```

### 知识迁移

```bash
# 迁移知识
POST /api/v1/agents/autonomous/knowledge/transfer
{
  "source_platform": "toutiao",
  "target_platform": "douyin",
  "knowledge_type": "best_practices"
}

# 获取迁移统计
GET /api/v1/agents/autonomous/knowledge/statistics
```

## 常见问题

### Q1: 为什么任务分解后不能自动执行？

**A**: 当前版本（Phase 2-3）主要实现了任务规划和智能体管理的基础架构。任务自动执行功能（Phase 4）需要为每种智能体类型实现具体的任务处理器，这是一个较大的工程，正在开发中。

### Q2: 如何判断应该注册哪些智能体？

**A**: 根据你的业务需求：
- **内容创作者**：至少需要 `research` + `content_generation` + `distribution`
- **多平台运营**：额外添加 `nurturing` 进行账号养护
- **高质量内容**：添加 `compliance_check` 和 `image_generation`

### Q3: 智能体的"负载"是什么意思？

**A**: 负载表示智能体当前的繁忙程度（0-100%）。当负载超过 80% 时，智能体被视为不可用，系统会选择其他空闲的智能体来执行任务。

### Q4: 知识迁移真的有效吗？

**A**: 是的！不同平台虽然有差异，但很多运营经验是相通的。例如：
- 黄金发布时间段
- 标题写作技巧
- 用户互动策略

系统会自动适配这些经验到目标平台的特性。
