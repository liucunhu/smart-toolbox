# 合规审查与爆款检测完整实现指南

## 🎯 核心原则

### 1. 合规审查 - 100%强制执行
- ✅ **不可绕过**：所有文章发布前必须通过合规审查
- ✅ **详细日志**：记录每一步检查过程
- ✅ **明确反馈**：指出具体违禁词位置和内容
- ✅ **立即终止**：发现违禁内容立即停止发布流程

### 2. 爆款检测 - 智能评估 + 事实核查
- ✅ **5维评分**：标题、内容、话题、互动、平台匹配度
- ✅ **热点核查**：不造谣、不虚构、不偏离事实、只描述事实
- ✅ **优化建议**：提供具体的改进方向
- ✅ **风险预警**：低潜力文章给出警告

### 3. 自动进化 - 基于数据持续优化
- ✅ **历史分析**：分析账号历史表现
- ✅ **策略调整**：根据数据优化生成策略
- ✅ **个性化推荐**：为每个账号定制主题

## 🔍 合规审查详解

### 检查内容

1. **标题违禁词检测**
   - 政治敏感词
   - 暴力恐怖词
   - 色情低俗词
   - 虚假广告词
   - 歧视性词汇

2. **正文违禁词检测**
   - 同上类别
   - 额外检查长文本中的隐藏违禁词

### 执行逻辑

```python
# 步骤 2.5: 合规审查（必须，100%执行）
logger.info(f"🔍 [步骤2.5/7] 正在进行合规审查（强制执行）...")
compliance_result = check_content_compliance(article_title, article_content, "toutiao")

if not compliance_result["passed"]:
    logger.error(f"❌ 合规审查失败: {compliance_result['error']}")
    logger.error("   ⛔ 文章包含违禁内容，禁止发布！")
    return {
        "status": "failed",
        "step": "compliance",
        "error": f"合规审查失败：{compliance_result['error']}。请修改内容后重新提交。"
    }

logger.info(f"✅ [步骤2.5/7] 合规审查通过")
```

## 🔥 爆款检测详解

### 5维评分体系

#### 1. 标题吸引力 (30分权重)

**评分标准**：
- ✅ 长度适中 (20-35字): +15分
- ✅ 包含数字: +10分
- ✅ 包含痛点/疑问词: +10分
- ✅ 包含情感词: +5分

**示例**：
```
标题："特朗普访华带两大'神秘人物'，鲁比奥赫格塞思的真实意图"
得分：85/100
优势：
  ✅ 标题长度适中 (45字)
  ✅ 包含悬念词（"神秘"、"真实意图"）
待优化：
  💡 建议添加具体数字（如"2大神秘人物"）
```

#### 2. 内容质量 (25分权重)

**评分标准**：
- ✅ 长度适中 (1200-2500字): +15分
- ✅ 段落结构合理 (5-15段): +10分
- ✅ 包含小标题: +10分
- ✅ 有数据支撑: +5分

#### 3. 话题热度 (20分权重)

**评分标准**：
- ✅ 热点话题: 90分
- ✅ 高关注领域 (AI/科技/财经): 70分
- ⚪ 普通话题: 50分

#### 4. 互动潜力 (15分权重)

**评分标准**：
- ✅ 包含问题引导: +15分
- ✅ 包含争议观点: +10分
- ✅ 包含行动号召: +10分

#### 5. 平台匹配度 (10分权重)

**头条偏好**：
- 深度分析
- 实用价值
- 数据支撑

### 热点话题事实核查（强制执行）

#### 四大原则

1. **不造谣** - 禁止绝对化表述
   ```python
   exaggeration_words = ["史上最", "绝对", "百分百", "必定", "必然", "100%"]
   ```

2. **不虚构** - 标注信息来源
   ```python
   # 检查是否包含"据说"、"传闻"等未证实信息
   unverified_patterns = [r"据说", r"传闻", r"网传"]
   ```

3. **不偏离事实** - 准确描述已知信息
   ```python
   # 检查数据是否有来源标注
   if re.search(r'\d+%', content) and "据" not in content:
       warnings.append("💡 包含数据但未标注来源")
   ```

4. **只描述事实** - 避免主观臆测
   ```python
   # 检查主观观点数量
   opinion_words = ["我认为", "我觉得", "应该", "必须"]
   if len(fact_sentences) > 3:
       warnings.append("💡 包含较多主观观点，热点话题应以事实为主")
   ```

#### 核查失败处理

```python
if not fact_check['passed']:
    logger.error(f"❌ 热点话题事实核查失败！")
    for issue in fact_check.get('issues', []):
        logger.error(f"   {issue}")
    
    return {
        "status": "failed",
        "step": "fact_check",
        "error": "热点话题事实核查失败：存在事实风险。请确保内容真实、客观、有来源。"
    }
```

## 🤖 自动进化功能

### 实现位置

`app/services/analytics/adaptive_content_evolver.py`

### 工作原理

1. **数据收集**
   - 每次发布后保存 `PublishRecord`
   - 记录文章标题、内容、分类、标签
   - 后续可集成阅读数、点赞数等指标

2. **智能分析**
   ```python
   # 分析历史表现最好的主题类型
   best_performing_keywords = []
   for record in publish_records:
       if record.content_task and record.content_task.article_title:
           title = record.content_task.article_title
           if len(title) > 10:
               best_performing_keywords.append(title[:20])
   ```

3. **个性化推荐**
   ```python
   # 获取个性化推荐（基于账号历史数据）
   recommendations = recommender.get_personalized_recommendations(account_id, count=3)
   
   if recommendations:
       best_recommendation = recommendations[0]
       final_topic = best_recommendation.get("topic", "")
       reason = best_recommendation.get("reason", "")
       confidence = best_recommendation.get("confidence", 0)
       
       logger.info(f"✅ [步骤2.0] 自动推荐主题: {final_topic}")
       logger.info(f"   推荐理由: {reason}")
       logger.info(f"   置信度: {confidence:.0%}")
   ```

4. **提示词优化**
   ```python
   # 获取智能分析结果（如果可用）
   if cache_service.is_analysis_available(account_id):
       optimized_prompt = cache_service.get_optimized_prompt(account_id)
       if optimized_prompt:
           logger.info(f"✅ [步骤2.1] 获取到账号 {account_id} 的智能分析优化提示词")
   ```

### 进化循环

```
发布文章
    ↓
保存记录 (PublishRecord)
    ↓
数据分析 (AdaptiveContentEvolver)
    ├─ 高表现文章特征提取
    ├─ 标题长度优化
    ├─ 内容长度优化
    ├─ 最佳发布时间
    └─ 分类表现分析
    ↓
生成优化提示词
    ↓
缓存到 Redis (24小时)
    ↓
下次发布时应用优化提示词
    ↓
持续迭代优化
```
