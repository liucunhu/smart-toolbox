## 🎯 问题描述

用户反馈：**"今日头条没看到发布的记录"**

**根本原因**:
1. 前端页面查询的是 `publish_records` 表（空表）
2. 实际数据保存在 `content_tasks` 表中
3. 缺少查询 `content_tasks` 的 API 端点

---

## ✅ 解决方案

### 1. 后端 API 新增

#### API 1: 获取内容任务列表

**端点**: `GET /api/v1/content/tasks`

**功能**: 查询 `content_tasks` 表，支持平台和状态筛选

**参数**:
- `platform`: 平台筛选（可选）- toutiao/douyin/xiaohongshu/bilibili/kuaishou
- `status`: 状态筛选（可选）- completed/processing/pending/failed
- `skip`: 跳过数量（默认 0）
- `limit`: 返回数量（默认 20，最大 100）

**响应示例**:
```json
{
  "total": 2,
  "tasks": [
    {
      "id": 2,
      "task_id": "fa5d765c-e4ac-48f1-a5cc-781254f2116c",
      "original_topic": "如何用deepseek生成网络小说",
      "target_platform": "toutiao",
      "article_title": "",
      "article_content_length": 2034,
      "article_category": "科技",
      "tags": ["#AI写作 #副业赚钱"],
      "status": "completed",
      "created_at": "2026-05-03T07:30:23"
    }
  ]
}
```

## 🔧 技术细节

### 数据库表结构

**content_tasks 表**:
```sql
CREATE TABLE content_tasks (
    id INTEGER PRIMARY KEY AUTO_INCREMENT,
    task_id VARCHAR(64) UNIQUE,
    original_topic TEXT,
    target_platform ENUM('douyin', 'xiaohongshu', 'bilibili', 'toutiao'),
    article_title VARCHAR(500),
    article_content TEXT,
    article_category VARCHAR(100),
    tags JSON,
    status VARCHAR(20),
    created_at DATETIME
);
```

### API 路由映射

| 前端请求 | 后端路由 | 数据库表 |
|---------|---------|---------|
| `/api/v1/content/tasks` | `list_content_tasks()` | `content_tasks` |
| `/api/v1/content/tasks/{id}` | `get_content_task_detail()` | `content_tasks` |
| `/api/v1/publish/records` | `list_publish_records()` | `publish_records` |

## ⚠️ 注意事项

### 1. 两个不同的表

- **content_tasks**: 内容创作任务（当前使用）
  - 存储生成的内容和发布状态
  - 自动发布时保存
  
- **publish_records**: 分发记录（暂未使用）
  - 存储每个账号的发布详情
  - 需要手动创建记录

### 2. 标题可能为空

某些情况下（如第一次测试），`article_title` 可能为空。这是正常的，会显示主题作为替代。

### 3. 文章内容获取

点击"详情"按钮时会异步加载完整的文章内容，可能需要短暂等待。
