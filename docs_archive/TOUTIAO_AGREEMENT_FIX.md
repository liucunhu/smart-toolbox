## ✅ 修复方案

### 1. 添加自动勾选用户协议逻辑

在 `login_with_manual_input` 方法中，在填充账号密码之前，先尝试自动勾选用户协议。

**修改位置**: `app/services/publish/toutiao_publisher.py` Line 64-97

**实现代码**:
```python
# ★★★ 关键：自动勾选用户协议 ★★★
logger.info("正在查找并勾选用户协议...")
agreement_selectors = [
    'input[type="checkbox"]',
    'label:has-text("用户协议")',
    'label:has-text("我已阅读并同意")',
    'label:has-text("隐私政策")',
    '[class*="agree"] input[type="checkbox"]',
    '[class*="protocol"] input[type="checkbox"]',
    'input[name*="agree"]',
    'input[name*="protocol"]',
    '.agreement-checkbox input[type="checkbox"]'
]

agreement_checked = False
for selector in agreement_selectors:
    try:
        agreement_checkbox = await self.page.query_selector(selector)
        if agreement_checkbox:
            is_checked = await agreement_checkbox.is_checked()
            if not is_checked:
                await agreement_checkbox.check()
                logger.info("✅ 已自动勾选用户协议")
            else:
                logger.info("用户协议已勾选")
            agreement_checked = True
            break
    except Exception as e:
        logger.debug(f"尝试选择器 {selector} 失败: {e}")
        continue

if not agreement_checked:
    logger.warning("⚠️  未找到用户协议复选框，可能需要手动勾选")
```

## 🔍 技术实现细节

### 多选择器策略

使用多种CSS选择器来定位用户协议复选框，提高兼容性：

1. **通用复选框**: `input[type="checkbox"]`
2. **文本匹配**: `label:has-text("用户协议")`
3. **类名匹配**: `[class*="agree"] input[type="checkbox"]`
4. **名称匹配**: `input[name*="agree"]`

### 智能检测

- 检查复选框是否已经勾选
- 如果未勾选，执行勾选操作
- 验证勾选是否成功
- 记录详细日志便于调试

### 容错处理

- 如果所有选择器都失败，记录警告但不中断流程
- 允许用户手动勾选
- 提供清晰的提示信息

## 💡 使用说明

现在头条登录的完整流程是：

```
1. 打开头条登录页
   ↓
2. 切换到密码登录模式
   ↓
3. ⭐ 自动勾选用户协议（新增）
   ↓

   ↓

   ↓

   ↓

   ↓
8. 保存Cookie到数据库
```

### 如果自动勾选失败

如果由于页面结构变化导致自动勾选失败：

1. 系统会记录警告日志：`⚠️ 未找到用户协议复选框，可能需要手动勾选`
2. 浏览器窗口保持打开
3. 用户可以手动勾选用户协议
4. 然后手动点击登录
5. 系统会自动检测登录成功并保存Cookie
